What made us smile

  • Mike was able to use the AI docs search to find out how to configure most steps.

    • What was confusing, did you encounter any issues?

    • Mike, with the help of the AI search, was able to find where to configure the rule to tag the VIP customers.
      `On Invoice paid -> InvoiceCustomerLifetime revenueAmountgreater or equal to 10000 -> Add tag VIP`
      But invoices are not created for all transactions, so the event should be changed to `Transaction processed`.
      This isn't a bug but it's easy to see merchants doing this.
    • Capitalization is wrong in the segment names. Should be "VIP"
    • The AI recommended using deposit strategies to tag the VIP customers.
      We are unable to edit filters on deposit strategies. Seems like a bug.
    • Gateway account restrictions would be a perfect place to ensure that only VIP customers could use Skrill, but tags are not available in the filters, unfortunately.
    • Another way to do this is with the gateway account selected event, tags are available there.

    • It seems the best way (besides the gateway account restriction which doesn't work) would be the `ready to pay was requested` rule.
      We have so many ways to accomplish this, having docs on the best practices would be helpful.
    • Actually we noticed that the AI search is a great way to test if our docs cover the topics. When we have good docs, the AI is able to find the answers.
