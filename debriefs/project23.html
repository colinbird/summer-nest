What made us smile

  • We were able to configure the gateway account and simulate approved and declined transactions based on test cards from Checkout.com docs.

    • What was confusing, did you encounter any issues?

    • We don't log the HTTP status code (or method ie POST) by default in the gateway logs. This would be useful for debugging as initially we had the wrong API key configured and got a 401.
    • The 401 was logged as the gateway original code but this is specific to the Checkout.com integration.
    • There can be a lag between when the transaction happens and when the gateway log is shown. This can be confusing for a new developer integrating since it seems like there are no logs and they may not try reloading the transaction details screen after ~1 minute.
